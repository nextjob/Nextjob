Network Shares

Nextjob utilizes shares for transfering reports (CSV and PDF files) to client pc's (via Accuterm Scripts).  In order for these transfers to work correctly both the share name and local path must be known to the Nextjob Software.  The information is saved in the NETWORK_PATH record of the SYSTEM_FILE.  
NETWORK_PATH_REC<SYS_SERVER_REPORT_PATH > defines the local path of the share.
NETWORK_PATH_REC<SYS_SHARE_REPORT_PATH >    defines the share name less the Server Name.
Example:

NETWORK_PATH_REC<SYS_SERVER_REPORT_PATH > = "C:\QMACCOUNTS\NXT_TEMP"
NETWORK_PATH_REC<SYS_SHARE_REPORT_PATH > = "\NXT_TEMP"

The Share used for file transfer is \\servername\NXT_TEMP which is a share defined for local server file C:\QMACCOUNTS\NXT_TEMP.
The proper permissions must be set on share \\servername\NXT_TEMP to allow access to all client pc.  This may require removal of password control on shares assigned to "EVERYONE" unless the client pc's will operate with user names known to the Windows Server hosting QM.
