Item Detail



This tab's function is for order item detail entry and review. It includes customer part number, description, delivery schedule, pricing, and process description.

Item Selection Frame:

Field Descriptions:
 
Item Selection List (Grid) - Used to select the line item to make active..  User clicks the up or down scroll arrows until the desired item number is displayed. Clicking on the displayed item makes it active. 

Add Item - User selects to add a new item to the order.

Item Detail Frame:

Field Descriptions:

Part Number - Displays the part number for the active line item, and used to enter the part number for new line items (See Add Item Button).  For part numbers already defined in the system database, saved data will be used to populate rev, description, drawing, material and process steps (if defined). For new part numbers, the Part Maintenance Dialog will open allowing the user to create a new part number record.  

Rev - Used to display or enter part number revision.

Description -  Used to display or enter the part description.

Drawing - Used to display or enter part drawing number.

Material - Used to display or enter the material type for the part.

Price Ea.  - Price customer is charged for each item shipped, or a lot charge if so specified. A price value must be entered but may be zero.  If zerom, the system will prompt for a price at the time of invoicing.  Also price information will not be visible to users with insufficient privileges. 

SPC - Sales Product Code - Drop down list to select Sales Product Code for sales order item.

Order Priority - User enters the optional numeric priority (0 - 100) for the line item.  This number is used to place this line item on the priority scheduling report.

Ship Hold - User checks the ship hold box to prevent the item from being shipped.

Item Notes -  User enters any internal item notes in this box (these notes are not printed on shipping documents).

Item Schedule Detail Grid:

Line #  - Numeric schedule detail line number, maintained by the system.

Date Due - Customer specified due date for the line #.

Order Qty - Quantity customer requires for date due.

Date Shipped - Date this Line # was shipped, populated by the system via Shipping.

Qty Shipped - Quantity Shipped to the customer, entered from the Shipping tab.

Packslip - Packslip id created by system via Shipping.

Inv Date - Invoice Date populated by the system via Invoicing.

Inv Nbr - Invoice Number populated by the system via Invoicing.

Line Notes - Notes entered to be printed on shipping documents.

Buttons:

View - Displays image of Part Number (if defined via Par Master).

Part Master - Selecting this button results in the Part Maintenance Dialog box being displayed for the Part Number of the currently active order item.  Changes made to the Part Master record for the Part Number can be copied to the order item record after completion of the maintenance function.

Part Activity - Selecting the Activity button results in the Activity Dialog box being displayed for the Part Number of the currently active order item. 

Print Order Label  - Generate and Print Order Label on defined label printer.

Print Receiving Label - Generate and Print Receiving Label on defined label printer. 

    * Note -  Routines used to Print Order Label (ORDER.LBL)  and Print Receiving Label (RECEIVING.LBL) assume a label printer than understands  Datamax-O'Neil Programming Language (DPL).

Item File - User selects to save the currently active order item record, this will also result in saving changes made to fields on the  Order Tab.

Item Close - User selects to cancel edits made to active order item record.

Release Wo. - Release work order for the current sales order item. See Work Orders 

Item Process Description Frame:

Field Descriptions:

Item Certification / Process - Used to define Certifications or Process Steps followed in creating or processing the Part Number.  User selects from a drop down list of standard certification / process steps or enters a custom one.  Note: Certification / Process steps are originally created with the Part Maintenance Dialog, and saved in the part record.  At order entry the Certification / Process steps are copied to the order record. Changing the Certification / Process step of an order will NOT change the Certification / Process step on the part record!

Certs - Used to indicate that the Certification / Process Description is included on any certification created when shipping the line item.

Buttons:

Print - User selects to print a hardcopy of the Certification / Process Sheet.

Modify - User selects to modify the currently highlighted Certification / Process.

Delete - User selects to delete the currently highlighted Certification / Process.

Insert Before - User selects to insert a process step before the currently highlighted Certification / Process.

Insert After - User selects to insert a process step after the currently highlighted Certification / Process.




Note: changes to the fields: Rev, Drawing, Description, and Certification / Process Description results in the changes being saved in the system database. But changes made to these fields affect the current line item of the current order only.
