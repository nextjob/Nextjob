Equates

The following equates define system behavior (found in NXT.INCLUDES NXT.EQUATES)

* AT GUI ERROR LEVEL
* SEE AT PROGRAMMING GUIDE FOR INFO
EQU NXT.GUI.ERROR.LEVEL TO 1  ;* 1 - WARNINGS, 2 - COMMAND FAILURE, 3 - FATAL ERROR
*
* DEFINE IF WE ARE RUNNING PROCESS BASED SYSTEM OR WORK ORDER BASED
*
EQU NX_PROCESS_BASED TO 0
EQU NX_WORKORDER_BASED TO 1
EQU NX.PROCESS.OR.WORKORDER TO NX_WORKORDER_BASED
*
* FLAG TO SIGNAL SENDING OF EMAILS WHEN PACKSLIP CUT (GUISUB.SHIP)
*
EQU NX.EMAIL.SHIP.NOTIFICATION TO @TRUE
*
* FLAG TO PREVENT PACKSLIP / CERT / INVOICE PRINTING DURING TESTING
*
EQU NX.PRINT.ENABLE TO @TRUE
*
*  SHIPPING METHOD TYPE
*  PRESHIP - WE CUT PAPER WORK PRIOR TO ACTUAL SHIPPING, S/R PERSON MANUALLY ENTERS
*            QTY SHIPPED ON PAPER WORK AND OFFICE PERSONNEL UPDATES ORDER INFORMATION
*  DEMSHIP - DEMAND SHIPPING, WE CUT PAPER WORK AT TIME OF SHIPPING, SYSTEM UPDATES ORDER INFO
*            AT TIME OF SHIPPING
*** NOTE !!! BEFORE SHIPPING METHOD IS CHANGED, ALL PRESHIPPED ORDERS MUST BE UPDATE VIA OFFICE PERSONNEL
*            BEFORE THE SHIPPING METHOD IS CHANGED (THIS IS WHY ITS AN EQUATE AND NOT A 
*            SYSTEM_FILE SETTING!
EQU PRESHIP TO 1
EQU DEMSHIP TO 2
EQU NX_SHIP_METHOD TO DEMSHIP
*
* ACCOUNTING EQUATES
* NX_AR_ACTIVE -  DO WE USE THE NEXTJOB AR SYSTEM (TRUE / FALSE)
*                 IF SET TO FALSE, SALES RECORD IS WRITEN WITH SA_INVOICE_ID WITH ???
*                 IF SET TO TRUE, WE ACTAUL CREATE AN INVOICE DEPENDING ON HOW NX_QB_ACCOUNTING IS SET
*
* NX_QB_ACCOUNTING - WE USE QUICK BOOKS ACCOUNTING (TRUE / FALSE)
*
EQU NX_AR_ACTIVE TO @FALSE
EQU NX_QB_ACCOUNTING TO @TRUE
*
* PART PICTURES PCL CONVERTER INSTALLED? 
** MKDPLC IS FOUND AT http://www.spectracolorservices.com/
EQU NXT_MKDPCL TO @TRUE            ;* DO WE HAVE MKDPCL ?
